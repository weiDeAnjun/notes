GameManager脚本

一、GameManager 必须负责 的核心内容

1. 全局状态管理

跨场景共享的数据：分数、关卡进度、游戏难度、玩家设置（如音量、画质）等。

游戏生命周期状态：是否在运行、暂停、失败、通关等（例：isGameOver、isPaused 等状态变量）。举例：你的代码中管理 score（分数）、totalTime（倒计时）、selectMode（难度），这些都是典型的全局状态，属于 GameManager 的核心职责。

2. 游戏流程控制

统筹关键流程：游戏开始（初始化参数）、暂停 / 继续、失败 / 通关判定、场景切换（如从关卡切换到结算界面）。

跨场景逻辑衔接：新场景加载后初始化玩家位置、重置分数等（对应你代码中 OnSceneLoaded 事件的处理）。

3. 全局事件分发

作为事件枢纽，向其他系统传递关键信息（如得分变化、时间耗尽），避免系统间直接耦合。举例：当分数达标时，GameManager 触发 “通关事件”，通知 UI 系统显示胜利面板、通知音频系统播放胜利音效。

4. 核心管理器的统筹

初始化并持有全局单例的引用（如 AudioManager、UIManager、SaveManager），确保它们在游戏启动时就绪。

协调管理器间的协作（例：通知 AudioManager 根据当前难度调整 BGM 节奏）。

5. 持久化数据交互

调用存档 / 读档逻辑（如通过 SaveManager 保存最高分、已解锁关卡），但不直接处理文件读写细节（交给专门的存档管理器）。

二、GameManager 绝对不应该负责 的内容

1. 局部逻辑细节

具体实体的行为：玩家移动、敌人 AI、单个道具的触发逻辑（这些属于 Player、Enemy 等组件的职责）。

局部状态：某一关卡内的临时变量（如当前场景的敌人数量，应放在关卡专属的 LevelManager 中）。

2. UI 组件的直接操作

避免直接修改按钮、文本等 UI 元素（如 scoreText.text = "得分：100"），这会让 GameManager 与 UI 强耦合。正确做法：通过 UIManager 提供的接口（如 UIManager.Instance.UpdateScore(100)）间接更新 UI，GameManager 只负责 “通知分数变化”。

3. 具体资源的加载 / 卸载

不直接处理模型、纹理等资源的加载（交给 ResourceManager），只负责触发 “加载下一关资源” 的指令。

4. 重复的业务逻辑

避免包含可复用的工具类逻辑（如时间格式化、数据校验），这些应抽成独立的 Utils 工具类。

5. 子系统的具体实现

不负责音频播放细节（如播放哪段音效、音量调节），这些交给 AudioManager；

不负责物理碰撞的检测逻辑（交给 Physics 系统或碰撞体组件）。

三、一句话总结边界

GameManager 是 “决策者” 和 “协调者”，而非 “执行者”：

负责 “要不要做”（如 “分数达标，该触发通关了”），

不负责 “具体怎么做”（如 “通关面板的动画如何播放”）。

按这个原则设计，你的 GameManager 会保持简洁、可维护，避免随着项目扩大变成难以调试的 “万能脚本”。

 





* 挂载 xx脚本的 GameObject 被设置为激活状态（SetActive(true)）。此时，脚本的生命周期函数才可能执行：

    Awake()：当 GameObject 被激活且脚本首次加载时执行（仅执行一次），无论是否访问静态变量。

    Start()：在Awake()之后，第一次进入帧更新前执行（仅执行一次）。

    Update()等：只有 GameObject 保持激活状态，才会每帧执行。

     





* 一定要注意unity，有些函数是有时效的，比如awake、start，它们是脚本激活1次使用一次，如果脚本激活后没有销毁，那就只有这1次


还要注意脚本和脚本挂在对象的关系，对象加载，脚本才激活





* 一定注意游戏对象是否是场景独有，如果需要跨场景吗，最好做成预制体，否则必须加载游戏对象所在场景，从这个场景传递游戏对象才行

二维图形挤出

选择样条线（如矩形）后，在修改器列表中添加 “挤出” 修改器，通过 “数量” 参数设置厚度（如创建墙体）。

三维模型面挤出

进入 “可编辑多边形” 的 “面” 子层级，选择面后应用 “面挤出” 修改器，通过 “挤出高度” 控制拉伸深度。

二、核心参数与功能扩展

1. 挤出深度与分段控制

数量（Amount）：决定挤出的物理距离，正值向外、负值向内（如制作凹陷效果）。

分段（Segments）：控制挤出方向的细分数量，分段数越高，后续弯曲或变形时细节越丰富（如弯曲管道需足够分段）。

2. 封口与表面类型

封口选项

3ds Max 中，“封口始端 / 末端” 可封闭挤出两端，支持 “变形”（适合动画）或 “栅格”（适合平滑表面）两种封口模式。

从中心挤出：3ds Max 的 “面挤出” 修改器勾选此选项后，挤出将以面中心为基准向四周扩展，可用于创建倒角效果。

角点、平滑以及Beizer

* **角点**：无控制柄，纯尖锐转折，适合直线与直线的硬连接。
* **平滑：**无控制柄的自动平滑
* **Bezier 角点**：独立控制柄，尖角 + 两侧独立曲率，适合 “硬转折 + 软边缘” 的混合结构。
* **Bezier 平滑**：联动控制柄，无缝平滑过渡，适合纯曲线造型。
